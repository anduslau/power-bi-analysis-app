3. **"Root element is missing" error diagnosed** - When opening HealthMetrics_PbixProj.pbit in Power BI Desktop, System.Xml.XmlException occurs at `Microsoft.PowerBI.Modeling.Contracts.DataModel.FormatInformation.GetFromXml`.
   - **Root cause**: Suspected malformed XML in Format annotation values within column JSON files.
   - **Evidence**: String columns (e.g., `day_name.json`) have `"value": ""` (empty string), while numeric/date columns have valid XML like `"value": "<Format Format=\"NumberWhole\" Accuracy=\"0\" />"`.
   - **Hypothesis**: Empty string `""` may cause XML parser to fail during Power BI's model upgrade process.

4. **Column JSON structure examined** - Generated by `generate_tables.py`, includes:
   - String columns: empty Format value
   - Numeric columns: `<Format Format=\"NumberWhole\" Accuracy=\"0\" />`
   - Date columns: `<Format Format=\"DateTimeShortDate\" />` or `<Format Format=\"DateTimeGeneral\" />`
   - `DataTypeAtRefresh` annotations include type marker (e.g., `"String#####not a type"`)

## Key things to remember

1. **Power BI Template (.pbit) serialization** - Power Query M expressions stored both in DataModelSchema (XML) and separate `.m` files. Must match for template to function correctly.

2. **Format annotation requirements** - XML values in column JSON must be well-formed or omitted. Empty string `""` appears to cause XML parser failures.

3. **Verification pattern** - Compare DataModelSchema expressions with `.m` files using string normalization (strip whitespace, normalize line endings).

4. **Pipeline state tracking** - Uses `state.json` to track last successful date, enabling incremental data collection on manual runs.
